# S-14: Bluesky

> AT Protocol · Кастомные ленты · Стартер-паки · Органический рост без алгоритма

## 1. Алгоритм и механика ленты

Архитектура Bluesky фундаментально отличается от любой другой соцсети:единого монолитного алгоритма нет. Вместо этого пользователи выбирают из подключаемых генераторов лент, построенных на открытом AT Protocol.

### Четыре типа лент

| Лента | Механика | Значение для роста |
| --- | --- | --- |
| Подписки | Чистая хронология, посты от подписок | НИЗКОЕ — охватывает только текущих подписчиков |
| Рекомендации | Алгоритмическая: скорость вовлечения, близость соц. графа, свежесть, определение языка | ВЫСОКОЕ — основной вектор роста для новых аккаунтов |
| Кастомные ленты | Сторонние алгоритмы: ключевые слова, порог вовлечения, списки авторов — любая логика | ОЧЕНЬ ВЫСОКОЕ — уникально для Bluesky, Тим может создать свою |
| Тренды | Быстрая скорость вовлечения в разделе "Обзор/Поиск" | СРЕДНЕЕ — зависит от новостного цикла |

```
СИГНАЛЫ РАНЖИРОВАНИЯ РЕКОМЕНДАЦИЙ:
  ┌─ Скорость вовлечения (20 лайков за 10 мин > 50 лайков за 6 ч)
  ├─ Ответы + цитирования весят > лайков (разговор = буст)
  ├─ Близость соц. графа (связи 2-й/3-й степени)
  ├─ Свежесть контента (сильный приоритет новизны)
  ├─ Определение языка (совпадает с предпочтениями пользователя)
  └─ Репутация аккаунта (кол-во подписчиков, соотношение вовлечения, возраст)
```

> **Инсайт:** Нет теневого бана:В Bluesky нет механизма теневого бана. Модерация использует лейблеры (сторонние сервисы маркировки), а не алгоритмическое подавление. Контент либо виден, либо удалён — никакой серой зоны.

### Архитектура AT Protocol

```
Приложение пользователя (Bluesky клиент)
    │
    v
Генератор ленты (любой сервер, реализующий спецификацию)
    │
    v
App View (индексирующий сервис Bluesky)
    │
    v
PDS (Персональный сервер данных — данные пользователя хранятся здесь)

КЛЮЧ: Тим может создать и хостить свой генератор ленты → позиционирование как авторитет
```

## 2. Кастомные ленты и стартер-паки

### Кастомные ленты: суперсила Bluesky

Любой разработчик может создавать алгоритмические ленты, используя спецификацию генератора лент AT Protocol. Ленты фильтруют публичный поток по ключевым словам, вовлечению, спискам авторов или любой другой логике. Пользователи могут закрепить ленты как вкладки.

```
НАСТРОЙКА ГЕНЕРАТОРА ЛЕНТЫ:
  git clone https://github.com/bluesky-social/feed-generator.git
  cd feed-generator && npm install
  # Настроить с паролем приложения Bluesky
  # Задеплоить на RUVDS через Docker
  # Стоимость: $0 (существующая инфраструктура)
```

### Стартер-паки: Главный хак роста

Курированные списки аккаунтов, на которые новые пользователи могут подписаться одним кликом. Хорошо позиционированный стартер-пак может принести50-500+ подписчиков за одну волну.

| Стартер-пак | Содержание | Стратегия распространения |
| --- | --- | --- |
| «AI-билдеры и архитекторы мульти-агентов» | 30-50 аккаунтов: AI-исследователи, создатели агентного AI, создатели инструментов, dev-адвокаты | Распространить в LinkedIn, Telegram, Dev.to — обновлять ежемесячно |
| «Карьера в технологиях и рынок труда» | Карьерные коучи, основатели джоб-бордов, HR-технологи, рекрутёры на Bluesky | Позиционирование СБОРКИ — шерить в карьерных сообществах |

> **Инсайт:** Механизм роста:Создание популярного стартер-пака позиционирует вас как лидера сообщества. Попадание в чужие стартер-паки — самый быстрый путь к росту подписчиков — цель: попасть в 5+ существующих AI/Tech паков.

## 4. Преимущества AT Protocol

Переносимость данных:Все посты, подписки и профиль хранятся на вашем персональном сервере данных (PDS). Экспортируемы в любой момент. Если Bluesky закроется, ваш контент выживет. Инвестиции защищены.

Переносимость аккаунта:Идентичность — это DID (децентрализованный идентификатор). Перемещение между PDS-провайдерами без потери подписчиков. Кастомные домены работают как хендлы.

Открытый поток данных:Все публичные данные доступны в реальном времени. Создание аналитических дашбордов на RUVDS. Мониторинг упоминаний, вовлечения, активности конкурентов бесплатно.

Система лейблов:Децентрализованная модерация через лейблеры. Будущая возможность: создать лейблер «Качественный AI-контент» для маркировки качественных постов в нише.

### Доменный хендл: мощный ход для бренда

> **Инсайт:** Критическая рекомендация:Установитьsborka.workкак хендл Bluesky вместоtimzinin.bsky.social. Каждый пост, ответ и репост показывает ваш домен. Это эквивалент верифицированного бейджа (подтверждает владение доменом). Бесплатная узнаваемость бренда с каждым взаимодействием. AI/tech-сообщество уважает кастомные доменные хендлы.

```
НАСТРОЙКА ДОМЕННОГО ХЕНДЛА (15 мин):
  1. Добавить DNS TXT запись: _atproto.sborka.work → did=did:plc:xxxxx
  2. Обновить хендл в настройках Bluesky → Расширенные → Сменить хендл
  3. Теперь каждый пост показывает "sborka.work" как хендл

  Альтернатива: использовать timzinin.com для фокуса на личном бренде
  Можно сменить позже — не необратимо
```

## 6. Тон коммуникации и культурные нормы

Культура Bluesky сформирована журналистами, учёными, разработчиками и людьми, ушедшими с X. Понимание этого критично для резонанса контента.

| Культурная норма | Значение для Тима |
| --- | --- |
| Искренность важнее иронии | Неподдельный энтузиазм по поводу работы заходит лучше, чем саркастические комментарии |
| Экспертиза уважается | Реальный опыт Тима с AI-системами будет оценён, а не высмеян |
| Самопиар зависит от контекста | «Посмотрите мой продукт!» = игнор. «Я сделал X, вот что узнал» = приветствуется |
| Alt-текст обязателен | ВСЕГДА добавлять alt-текст к изображениям. Культурная норма, не опция |
| Анти-хасл культура | Представлять бизнес как мастерство, а не «работай на износ» |
| Высокая техническая грамотность | Не нужно упрощать. Аудитория понимает API, Docker, агенты |

> **Внимание:** Чего ИЗБЕГАТЬ:«Увеличьте продуктивность в 10 раз с AI» — хасл-культура / маркетинговый языкПриманка вовлечённости («Репост если согласны») — воспринимается как манипуляцияКросс-пост точной копии из Twitter — выглядит лениво, другой культурный контекстЖёсткие продажи — сопротивление сообщества чистому маркетингуПосты на русском — английский доминирует; русскоязычная аудитория на Bluesky минимальнаОтсутствие alt-текста — нарушение культурной нормы
