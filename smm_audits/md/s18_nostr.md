# S-18: Протокол Nostr

> Децентрализованная соцсеть через реле — Zap-микроплатежи, NIP-05 идентификация, Bitcoin-нативная аудитория, свой реле-сервер

## 1. Механика протокола Nostr

Nostr (Notes and Other Stuff Transmitted by Relays) — децентрализованный протокол, построенный на трёх примитивах:Клиенты(приложения),Реле(WebSocket-серверы) иКлючевые пары(криптографическая идентичность secp256k1, та же кривая что у Bitcoin). Без блокчейна, без токена, без центральной власти.

### Основная архитектура

```
Приватный ключ (nsec) →подписывает события→ Публичный ключ (npub) = ваша идентичность →NIP-05= человекочитаемый (tim@sborka.work)
```

### Основные типы событий

| Тип | Название | Релевантность |
| --- | --- | --- |
| 0 | Метаданные профиля | ВЫСОКАЯ — первое впечатление, имя, био, NIP-05, Lightning-адрес |
| 1 | Короткая текстовая заметка | ВЫСОКАЯ — ежедневные посты (как твиты, до ~65KB) |
| 6 | Репост | ВЫСОКАЯ — расшаривание чужих заметок (вовлечение) |
| 7 | Реакция | ВЫСОКАЯ — лайки, эмодзи-реакции |
| 9734/9735 | Запрос/Квитанция Zap | ВЫСОКАЯ — Lightning-микроплатежи на заметки |
| 10002 | Список реле (NIP-65) | ВЫСОКАЯ — указывает сети где искать ваш контент |
| 30023 | Лонг-форм контент | ВЫСОКАЯ — полноценные Markdown-статьи (блог-посты) |

### Модель распространения контента

> **Инсайт:** В отличие от централизованных платформ, ЕДИНОГО алгоритма нет. Каждый клиент реализует свою логику ленты. Primal имеет трендовые ленты, Damus более хронологический. Вашвыбор релеисетевые эффектыважнее, чем "обыгрывание алгоритма". Получить репост (kind 6) от кого-то с более широким покрытием реле — основной вирусный механизм.

> **Внимание:** События неизменяемы после публикации (без редактирования). Нет гарантированной доставки если реле упадут. Нет глобального поиска (зависит от реле). Картинки нужно хостить внешне (nostr.build, void.cat). Это компромиссы протокола ради цензуро-устойчивости.

### Интеграция пайплайна публикации

```
auto_publisher.py → подписать событие nsec →опубликовать на 5+ реле через WebSocket→ видно на ВСЕХ клиентах одновременно
```

Использоватьnostr-sdk(Python, на основе rust-nostr) для программной публикации событий. Без OAuth, без API-ключей, без лимитов от центральной власти. Просто подписать JSON и отправить через WebSocket.

## 5. Lightning Network и стратегия Zap

Zaps — мгновенные Bitcoin-микроплатежи, привязанные к конкретным заметкам — убойная фича, отличающая Nostr от любой другой соцсети.

### Экономика Zap

| Уровень | Сатоши | Эквивалент USD | Вызывается |
| --- | --- | --- | --- |
| Простая благодарность | 21-100 | $0.02-0.10 | Базовый хороший пост |
| Хороший пост | 500-1 000 | $0.50-1.00 | Полезный инсайт |
| Отличный контент | 2 100-10 000 | $2-10 | Глубокий технический пост |
| Исключительный / вирусный | 21 000-100 000 | $20-100 | Контент, определяющий категорию |
| "Zap-шторм" | 500 000+ | $500+ | Коллективная волна на вирусных постах |

### Настройка Lightning-адреса

### Стратегическая ценность Zaps

1. Социальное доказательство:Счётчики zap видны на постах и сигнализируют качество (как лайки, но с реальными деньгами)
2. Строительство сообщества:Zap другим строит отношения и видимость
3. Валидация спроса:Если AI-контент получает значимые zaps, это подтверждает рынок
4. Мост к сервисам:"Если вам нравится что делают мои агенты, представьте их работающими на ВАШ бизнес"

> **Внимание:** КРИТИЧЕСКИЕ культурные правила: говорить "сатоши" не "доллары". Говорить "Bitcoin" не "крипто". НИКОГДА не упоминать альткоины (Ethereum, Solana). Сначала zap, потом вовлечение. Круглые числа (21, 210, 2100) культурно значимы (Bitcoin-отсылки).
