## 1. Алгоритм и лента Dev.to (2026)

Dev.to построен на open-source платформеForem. Алгоритм ранжирования доступен в коде:app/services/articles/feeds/weighted_query_strategy.rb.

### Три режима ленты

| Лента | Механика | Значение |
| --- | --- | --- |
| Relevant(по умолч.) | Персонализация по тегам + авторам + свежесть | Основной источник трафика для залогиненных |
| Latest | Чистая хронология | 15-30 минут видимости, окно для новых авторов |
| Top | Реакции + комментарии за период | Вечнозелёный трафик, цель для серий |

### Формула ранжирования (упрощённая)

```
score = (positive_reactions_count × 1.0)+ (comments_count × 0.7)+ (page_views × 0.0001)+ (recency_factor × 0.5)+ (user_following_boost × 1.5)+ (tag_following_boost × 0.3)
```

### Сигналы ранжирования

| Сигнал | Вес | Как работает |
| --- | --- | --- |
| Реакции (сердечки/единороги/закладки) | ВЫСОКИЙ | Основной сигнал. Единороги и закладки весят чуть больше сердечек |
| Комментарии | ВЫСОКИЙ | Уникальные комментаторы > множество комментов от одного человека |
| Время чтения / дочитывание | СРЕДНИЙ | 5-10 мин оптимально. <2 мин — мало. >15 мин — низкое дочитывание |
| Свежесть | СРЕДНИЙ | Период полураспада ~48 часов. Пик день 1-2, затем затухание |
| Авторитет автора | СРЕДНИЙ | Больше подписчиков + постоянная активность = бонус в Relevant |
| Популярность тега | НИЗКО-СРЕДНИЙ | Статьи в популярных тегах (javascript 150K+, ai 50K+) = больше показов |

> **Инсайт:** Ключевой инсайт:Скорость вовлечённости решает. 5+ реакций за первые 2 часа значительно важнее 50 реакций за неделю. Алгоритм награждает быстрый старт.

### Воронка серий — самая эффективная стратегия

```
Статья 1: "Why Multi-Agent AI" (осведомлённость) → Подпишитесь для Части 2Статья 2: "My 6-Agent Architecture" (интерес) → Часть 3 + GitHub-репоСтатья 3: "Building Agent #1" (желание) → Часть 4 + репо с кодомСтатья 4: "Monitoring, Costs, Failures" (доверие) → Часть 5 + ссылка на СБОРКУСтатья 5: "Career Implications" (действие) → sborka.work + подписка + GitHub
```

К Статье 5 читатель получил настоящую ценность из 4 предыдущих. CTA на sborka.work заслужен, не навязан.

### Чего НЕ делать

> **Внимание:** НЕ начинать/заканчивать каждую статью с "Загляните в СБОРКУ!"НЕ ставить CTA в первый параграф — читатели отвалятсяНЕ использовать срочность ("Мест мало!", "Подпишитесь СЕЙЧАС!")НЕ ставить ссылку на sborka.work чаще 2-3 раз за статьюНЕ делать CTA целью статьи — сначала ценность
