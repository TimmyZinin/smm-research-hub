2. Форматы контента

2.1 Главы (поддерживаются Apple)

Apple Podcasts отображает заголовки глав в UI плеера. Очень мало русских подкастов используют главы — это дифференциатор.

Глава 1: 00:00 - Интро (проблемный хук)
Глава 2: 00:30 - Основная проблема
Глава 3: 06:00 - Практические шаги
Глава 4: 11:00 - План действий + CTA

2.2 Описание эпизода (HTML)

Apple рендерит ограниченный HTML: <p>, <a href>, <b>, <ul>/<li>. Ссылки кликабельны — это основной CTA-канал.

2.3 Обложка эпизода

Apple поддерживает уникальную обложку для каждого эпизода (3000x3000). Рекомендация: шаблон с брендом + номер + ключевое слово. Генерация через Python (Pillow) или Gemini.

2.4 Структура сезонов

СезонТемаЭпизодыФормат
Сезон 1"Начало пути"ep01-ep08Монолог
Сезон 2"Реальные истории"ep09-ep16Интервью
Сезон 3"Продвинутые стратегии"ep17-ep24Микс

Премьеры сезонов получают алгоритмический буст (аналог нового шоу). Создаёт естественные контентные арки.

4. Apple Podcasts Connect

4.1 Получение доступа

Получить доступ через podcastsconnect.apple.com → Apple ID Тима → Отправить RSS cloud.mave.digital/69800 → Подтверждение по email → Доступ к аналитике + метаданным.

Без получения доступа = НОЛЬ аналитики. Нельзя редактировать метаданные, отвечать на отзывы, подать на редакцию.

5.2 Адаптации

  • Начало: Проблема в первую очередь (правило ToV #1). Аудитория Apple ценит конкретику
  • Глубина: Чуть глубже, чем для Spotify. 14 мин позволяют
  • Полировка: Стабильное интро/аутро, плавные переходы — аудитория Apple замечает
  • Тон CTA: Тёплый: "Если выпуск помог, буду благодарен за оценку" — не "СТАВЬТЕ 5 ЗВЁЗД"
  • Говори "follow", не "subscribe" — Apple сменил термин в 2021

5.3 Качество звука

Нормализация: ffmpeg -i input.mp3 -af loudnorm=I=-16:TP=-1.5 output.mp3

ElevenLabs = чистый звук без шумов, кликов, дыхания. Фактически лучше большинства подкастов, записанных людьми.

5 методов получения отзывов (соответствуют ToV)

  1. Просьба в эпизоде — в конце, после ценности. "Буду благодарен за оценку" (каждый 3-4 эпизод)
  2. Telegram-пост — после нового эпизода в @public_sborka с Apple-ссылкой
  3. Вебинар — упоминание после ивента (тёплая аудитория)
  4. Бот — одноразовое сообщение после онбординга
  5. Взаимные — взаимные отзывы с другими RU-подкастерами

Скрипт для запроса отзыва

"Если этот выпуск оказался полезным, я буду благодарен за оценку в Apple Podcasts. Это занимает буквально минуту, а нам помогает находить тех, кому подкаст нужен. Просто откройте СБОРКУ в приложении Подкасты и нажмите на звёздочки. Спасибо." (~15 сек)

Чего НЕ делать

  • Не стимулировать отзывы (скидки за отзыв — Apple удалит)
  • Не просить "5 звёзд" — просить "честную оценку"
  • Не создавать фейковые отзывы
  • Не просить в каждом эпизоде — ротация каждые 3-4
