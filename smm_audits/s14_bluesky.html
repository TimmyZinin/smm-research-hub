1. Алгоритм и механика ленты

Архитектура Bluesky фундаментально отличается от любой другой соцсети: единого монолитного алгоритма нет. Вместо этого пользователи выбирают из подключаемых генераторов лент, построенных на открытом AT Protocol.

Критический инсайт: Bluesky НЕ штрафует за внешние ссылки. В отличие от X/Twitter, который подавляет посты со ссылками, Bluesky обращается с постами со ссылками так же, как с текстовыми. Это самое большое преимущество CTA среди всех платформ.

Нет теневого бана: В Bluesky нет механизма теневого бана. Модерация использует лейблеры (сторонние сервисы маркировки), а не алгоритмическое подавление. Контент либо виден, либо удалён — никакой серой зоны.

Архитектура AT Protocol

Приложение пользователя (Bluesky клиент) │ v Генератор ленты (любой сервер, реализующий спецификацию) │ v App View (индексирующий сервис Bluesky) │ v PDS (Персональный сервер данных — данные пользователя хранятся здесь) КЛЮЧ: Тим может создать и хостить свой генератор ленты → позиционирование как авторитет

2. Кастомные ленты и стартер-паки

Кастомные ленты: суперсила Bluesky

Любой разработчик может создавать алгоритмические ленты, используя спецификацию генератора лент AT Protocol. Ленты фильтруют публичный поток по ключевым словам, вовлечению, спискам авторов или любой другой логике. Пользователи могут закрепить ленты как вкладки.

Стратегия: Создать кастомную ленту «AI-агенты и мульти-агентные системы». Фильтрация потока по: «AI agent», «multi-agent», «CrewAI», «AutoGen», «LangGraph», «agentic AI». Хостинг на RUVDS (Docker-контейнер, стоимость $0). Это позиционирует Тима как куратора/авторитета в нише AI-агентов.
НАСТРОЙКА ГЕНЕРАТОРА ЛЕНТЫ: git clone https://github.com/bluesky-social/feed-generator.git cd feed-generator && npm install # Настроить с паролем приложения Bluesky # Задеплоить на RUVDS через Docker # Стоимость: $0 (существующая инфраструктура)

Стартер-паки: Главный хак роста

Курированные списки аккаунтов, на которые новые пользователи могут подписаться одним кликом. Хорошо позиционированный стартер-пак может принести 50-500+ подписчиков за одну волну.

Механизм роста: Создание популярного стартер-пака позиционирует вас как лидера сообщества. Попадание в чужие стартер-паки — самый быстрый путь к росту подписчиков — цель: попасть в 5+ существующих AI/Tech паков.

4. Преимущества AT Protocol

Переносимость данных: Все посты, подписки и профиль хранятся на вашем персональном сервере данных (PDS). Экспортируемы в любой момент. Если Bluesky закроется, ваш контент выживет. Инвестиции защищены.
Переносимость аккаунта: Идентичность — это DID (децентрализованный идентификатор). Перемещение между PDS-провайдерами без потери подписчиков. Кастомные домены работают как хендлы.
Открытый поток данных: Все публичные данные доступны в реальном времени. Создание аналитических дашбордов на RUVDS. Мониторинг упоминаний, вовлечения, активности конкурентов бесплатно.
Система лейблов: Децентрализованная модерация через лейблеры. Будущая возможность: создать лейблер «Качественный AI-контент» для маркировки качественных постов в нише.

Доменный хендл: мощный ход для бренда

Критическая рекомендация: Установить sborka.work как хендл Bluesky вместо timzinin.bsky.social. Каждый пост, ответ и репост показывает ваш домен. Это эквивалент верифицированного бейджа (подтверждает владение доменом). Бесплатная узнаваемость бренда с каждым взаимодействием. AI/tech-сообщество уважает кастомные доменные хендлы.
НАСТРОЙКА ДОМЕННОГО ХЕНДЛА (15 мин): 1. Добавить DNS TXT запись: _atproto.sborka.work → did=did:plc:xxxxx 2. Обновить хендл в настройках Bluesky → Расширенные → Сменить хендл 3. Теперь каждый пост показывает "sborka.work" как хендл Альтернатива: использовать timzinin.com для фокуса на личном бренде Можно сменить позже — не необратимо

6. Тон коммуникации и культурные нормы

Культура Bluesky сформирована журналистами, учёными, разработчиками и людьми, ушедшими с X. Понимание этого критично для резонанса контента.

Экспертиза уважаетсяРеальный опыт Тима с AI-системами будет оценён, а не высмеян
Самопиар зависит от контекста«Посмотрите мой продукт!» = игнор. «Я сделал X, вот что узнал» = приветствуется
Alt-текст обязателенВСЕГДА добавлять alt-текст к изображениям. Культурная норма, не опция
Анти-хасл культураПредставлять бизнес как мастерство, а не «работай на износ»

Чего ИЗБЕГАТЬ:
  • «Увеличьте продуктивность в 10 раз с AI» — хасл-культура / маркетинговый язык
  • Приманка вовлечённости («Репост если согласны») — воспринимается как манипуляция
  • Кросс-пост точной копии из Twitter — выглядит лениво, другой культурный контекст
